# Security Policy

Thank you for helping keep this project secure. We take security seriously and appreciate your efforts to responsibly disclose vulnerabilities.

## Supported Versions

Only the latest versions of the main branch are actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Older | :x:                |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security bugs or vulnerabilities.**

If you discover a security vulnerability, please report it:

 **GitHub Private Vulnerability Reporting:** Please click the following: [link](https://github.com/Owenb135/THE-MULTIVERSE/security/advisories)

### What to Include
To help patch the issue quickly, please include:
* A detailed description of the vulnerability.
* Clear steps, scripts, or instructions to reproduce the issue.
* Potential impact (e.g., remote code execution, crash, data leak).

## Our Response Process

* **Acknowledgment:** You will receive an acknowledgment of your report within 48 hours.
* **Investigation:** We will investigate the issue and determine the appropriate fix.
* **Resolution:** A patch will be developed, tested, and pushed to the repository. We aim to resolve reported issues within 30 days of acknowledgment.

## Disclosure Strategy

We follow responsible disclosure principles. We ask that you do not publish details about the vulnerability until a patch has been released and users have had a reasonable timeframe to update their systems.
